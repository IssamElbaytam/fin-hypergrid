Interface: dataSourceHelperAPI

dataSourceHelperAPI

Hypergrid's data source objects transform data row collections (for example, by indexing them). These might include filter data sources that hide data rows and sorter data sources that rearrange data rows.

When such a data source is found in the transformation pipeline, Hypergrid sets its api property with a reference to an object that implements this interface.

Later, the transformations are executed (by applyAnalytics). The apply method of each data source is called in turn. A filtering data source's apply method calls this API's test method, once for each data row (typically to build an index of included rows).

Methods

properties(properties)

Column property accessor.

There are two types of calls:

  • "Getter" call - when properties.getPropName is defined
  • "Setter" call - when properties.getPropName is not defined

For "setter" calls only: Assigns the values of all enumerable members of properties to the filter properties with the same names, with the following exceptions:

  • Value is undefined - The filter property is deleted instead of being assigned to the filter property.
  • Value is a function - The function is only assigned to the filter when the property name appears in this.firstClassProperties. Otherwise, the function is called and the returned value is assigned to the filter instead.

Note: column and getPropName are reserved names in the properties namespace because they have special meaning in the properties object passed to this function. Filter developers should not implement properties with these names.

Parameters:
Name Type Description
properties object

Contains property values to assign to the filter. This object is required. However, none of the following options are required.

Properties
Name Type Attributes Description
column object <optional>

If defined, value(s) pertain to the indicated column only. If omitted, value(s) pertain to the whole filter.

Note that dataModels.JSON~propPrep defines column as non-enumerable. This makes it easy for your helper API's properties method implementation to ignore it. .

Properties
Name Type Attributes Description
index object <optional>

Column index as defined by dataSource.getFields().
name object <optional>

Column name as defined in dataSource.getFields().
getPropName string <optional>

"Get" (return) the value of the property with the supplied name. If undefined,return null. (All other members of properties are ignored.)
Returns:

One of:

  • Property value when getPropName defined.
  • null when getPropName undefined.
  • undefined when setting property value(s).

Implementation of this method is required.
